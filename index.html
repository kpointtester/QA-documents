QA Documentation Repository

This repository contains QA documentation including feature specifications, test cases, and automation scripts.

Prompt: Generate QA Documentation from Discussion Transcript

Role

You are a Senior QA Engineer. Your task is to transform a raw product discussion into clear, testable, and professional QA documentation.

Objective

Based only on the provided transcript, generate a full QA deliverable package for a new feature. Do not assume requirements not present or logically implied in the source.

Inputs

  • Feature Name: [Provided separately]
  • Discussion Transcript: [Provided separately]

Templates

  1. Feature Spec Template (Feature Spec Template 2025.docx)
  2. Gherkin Automation Format Example

Required Deliverables

1. Feature Specification Document

  • Follow the structure of the provided template exactly
  • All content must be derived from the transcript

2. Manual Test Case Suite

Format: Table with columns:

  • Test Case ID
  • Title
  • Prerequisites
  • Steps
  • Expected Result

Test Categories:

[Test categories list with detailed descriptions]

3. BDD Automation Feature File

Format: Gherkin (.feature)

Requirements:

  • @tags for logical scenario grouping
  • Scenario Outline + Examples for data-driven tests
  • Shared Background steps
  • Clear readability for all stakeholders
  • Conformance to Gherkin template

Constraints

  1. Use only the transcript as your source
  2. Do not invent requirements—only derive or logically clarify
  3. All output must be suitable for direct use by QA, dev, and product teams
